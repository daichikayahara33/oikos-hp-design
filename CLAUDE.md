# OIKOS HP - NPO Homepage Project

## Project Overview
NPOのホームページを制作するプロジェクト。

## Organization Structure

```
                    [PM]
           プロジェクトマネージャー
          /         |         \
    [Design]    [Engineering]  [Content]
    デザイン部    エンジニア部    コンテンツ部
                    |
              [QA / Deploy]
              品質・運用部
```

## Roles & Responsibilities

### PM（プロジェクトマネージャー）
- **担当**: プロジェクト全体の統括
- **権限**: 最終意思決定、タスク割り振り、進捗管理
- **方針**:
  - ユーザーの要望をヒアリングし、各部門へ指示を出す
  - 各部門の成果物をレビューし、品質を担保する
  - スコープの管理（不要な機能追加を防ぐ）
- **使用エージェント**: Plan エージェント

### デザイン部（UI/UX Designer）
- **担当**: ビジュアルデザイン、レイアウト設計
- **責務**:
  - ページレイアウト・ワイヤーフレーム設計
  - カラーパレット・タイポグラフィの決定
  - レスポンシブ対応の設計
  - NPOとしての信頼感・親しみやすさの表現
- **使用エージェント**: 汎用エージェント（design指示）

### エンジニア部（Frontend Engineer）
- **担当**: 実装全般
- **責務**:
  - 技術スタック選定・環境構築
  - コンポーネント実装
  - アニメーション・インタラクション
  - パフォーマンス最適化
- **使用エージェント**: 汎用エージェント（engineering指示）

### コンテンツ部（Content Strategist）
- **担当**: コンテンツ企画・制作
- **責務**:
  - ページ構成の企画
  - コピーライティング（見出し・本文）
  - SEO対策・メタ情報設計
  - トーン&マナーの統一
- **使用エージェント**: 汎用エージェント（content指示）

### 品質・運用部（QA / DevOps）
- **担当**: 品質保証とデプロイ
- **責務**:
  - アクセシビリティチェック
  - ブラウザ互換性テスト
  - ホスティング設定・デプロイ
  - CI/CD構築
- **使用エージェント**: 汎用エージェント（qa指示）

## Workflow

### フェーズ1: 企画・設計
1. PM がユーザーと要件を確認
2. コンテンツ部 がページ構成を提案
3. デザイン部 がレイアウト・デザイン方針を策定
4. PM が承認

### フェーズ2: 環境構築
1. エンジニア部 が技術スタック選定・プロジェクト初期化
2. PM がレビュー・承認

### フェーズ3: 実装
1. エンジニア部 がコンポーネントを実装
2. デザイン部 がスタイリングを指示
3. コンテンツ部 がテキスト・画像を提供
4. PM が各成果物をレビュー

### フェーズ4: 品質保証・公開
1. 品質・運用部 がテスト・チェック
2. 品質・運用部 がデプロイ設定
3. PM が最終確認・公開承認
